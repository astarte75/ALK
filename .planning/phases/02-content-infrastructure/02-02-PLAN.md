3. Create `scripts/lib/image-upload.ts`:
   - Function `uploadImage(env, imageUrl, title)` that:
     1. Strips WordPress thumbnail suffix (`-\d+x\d+\.` → `.`) for max resolution
     2. Tries fetching full-res URL; falls back to original URL on 404
     3. Downloads image via native `fetch`
     4. Creates upload via `env.createUpload({ file: buffer })`
     5. Creates asset with `uploadFrom` link in `it-IT` locale (non-localized assets)
     6. Calls `asset.processForAllLocales()`
     7. Polls until `fields.file['it-IT'].url` is defined (max 30 retries, 1s interval)
     8. Publishes asset, returns asset ID
   - Function `uploadAllImages(env, imageUrls)` that:
     1. Processes images sequentially (not parallel — CMA rate limit)
     2. Returns `Map<string, string>` (originalUrl → assetId)
     3. Logs progress: "Uploading image 5/42: ermes-logo.png"
     4. On download failure: logs warning and skips (returns null for that URL)
   - Handle image deduplication: if same URL already uploaded, return existing asset ID

4. Create `scripts/lib/rich-text.ts`:
   - Function `textToRichText(text: string)` — converts plain text to Contentful Rich Text document
   - Function `markdownToRichText(markdown: string)` — converts markdown to Rich Text:
     - Parse paragraphs (split by double newline)
     - Parse headings (# ## ###)
     - Parse bold (**text**) and italic (*text*)
     - Parse links [text](url)
     - Parse unordered lists (- item)
     - Wrap everything in proper Document > Block > Text node structure
   - Handle edge cases: empty strings → empty document, null → empty document
   - Output must match Contentful Rich Text JSON schema exactly (nodeType, data, content, marks arrays)

5. Create `scripts/lib/scraper.ts`:
   - Function `scrapeNewsArticle(url: string)` that:
     1. Fetches the full HTML page from the URL
     2. Extracts the main article content (from the post body)
     3. Returns `{ title, date, body, images }` with body as markdown
   - Handle both IT and EN URLs (check if /en/ version exists)
   - On fetch failure: return null with warning log

6. Create `scripts/migrate.ts` — main orchestrator:
   ```
   async function main() {
     const args = process.argv.slice(2)
     const dryRun = args.includes('--dry-run')
     const skipModels = args.includes('--skip-models')
     const skipImages = args.includes('--skip-images')
     const skipEntries = args.includes('--skip-entries')

     // Connect to Contentful
     const client = createClient({ accessToken: CONTENTFUL_MANAGEMENT_TOKEN })
     const space = await client.getSpace(CONTENTFUL_SPACE_ID)
     const env = await space.getEnvironment('master')

     // Step 1: Create content types
     if (!skipModels) await createContentTypes(env)

     // Step 2: Upload images
     let assetMap = new Map()
     if (!skipImages) assetMap = await uploadAllImages(env, collectImageUrls())

     // Step 3: Create entries
     if (!skipEntries) {
       await createOffices(env)
       await createInvestmentPlatforms(env)
       await createFunds(env)
       await createPortfolioCompanies(env, assetMap)
       await createTeamMembers(env, assetMap)
       await createNewsArticles(env, assetMap)
       await createPages(env)
       await createSiteConfig(env, assetMap)
     }

     console.log('Migration complete!')
   }
   ```
   - Supports `--dry-run` flag: logs all planned operations without calling Contentful API
   - Supports `--skip-models`, `--skip-images`, `--skip-entries` for partial runs
   - Entry creation order matters: offices → platforms → funds → companies (needs fund refs) → team → news → pages → config (needs office refs)
