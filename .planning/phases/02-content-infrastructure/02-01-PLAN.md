1. Create `src/lib/contentful/types.ts` with all 8 EntrySkeletonType definitions following RESEARCH.md Pattern 2:
   - `PortfolioCompanySkeleton`: name, slug, description (RichText), shortDescription (Text), sector, investmentType, year, logo (AssetLink), website, featured (Boolean), fundRef (EntryLink to fund), sortOrder
   - `TeamMemberSkeleton`: name, slug, role, category, bio (RichText), photo (AssetLink), linkedIn, isBoard (Boolean), sortOrder
   - `FundSkeleton`: name, slug, description (RichText), strategy, status, fundSize, investmentPeriod, platformRef (EntryLink to investmentPlatform)
   - `NewsArticleSkeleton`: title, slug, date (Date), category, excerpt, body (RichText), featuredImage (AssetLink), externalUrl
   - `InvestmentPlatformSkeleton`: name, slug, description (RichText), strategy, funds (Array of EntryLink to fund)
   - `PageSkeleton`: title, slug, body (RichText), sections (Object — JSON for structured page sections)
   - `SiteConfigSkeleton`: siteName, tagline, heroVideoUrl, logo (AssetLink), socialLinkedIn, socialTwitter, contactEmail, offices (Array of EntryLink to office)
   - `OfficeSkeleton`: city, address, phone, isHeadquarters (Boolean), sortOrder
   - Export resolved entry types: `PortfolioCompany`, `TeamMember`, `Fund`, `NewsArticle`, `InvestmentPlatform`, `Page`, `SiteConfig`, `Office`

   Field localization rules:
   - Localized (both IT + EN): name, title, description, bio, body, excerpt, tagline, role, sections, shortDescription
   - NOT localized (IT-only key): slug, year, investmentType, sector, category, website, linkedIn, phone, address, sortOrder, isBoard, featured, isHeadquarters, strategy, status, fundSize, investmentPeriod, date, externalUrl, heroVideoUrl, socialLinkedIn, socialTwitter, contactEmail, city
   - Asset links: NOT localized (same image both locales)
   - Entry links: NOT localized (same references both locales)

2. Create `src/lib/contentful/client.ts` following RESEARCH.md Pattern 1:
   - Export `contentfulClient` (delivery), `contentfulPreviewClient` (preview)
   - Export `getClient(preview?: boolean)` factory function
   - Use `process.env.CONTENTFUL_SPACE_ID!` etc. (server-only, no NEXT_PUBLIC_)

3. Create `src/lib/contentful/locale.ts` following RESEARCH.md Pattern 3:
   - `LOCALE_MAP`: `{ it: 'it-IT', en: 'en-US' }`
   - Export `toContentfulLocale(routeLocale: string): string`

4. Create `src/lib/contentful/fetchers.ts` following RESEARCH.md Pattern 4:
   - Wrap all fetchers in `unstable_cache` with content-type-specific tags for ISR
   - Each function accepts `locale: string` and optional `preview: boolean`
   - Functions to create:
     - `getPortfolioCompanies(locale)` → `PortfolioCompany[]` — ordered by sortOrder, include 2 for fund reference, tag: 'portfolioCompany'
     - `getPortfolioCompanyBySlug(slug, locale)` → `PortfolioCompany | null` — tag: 'portfolioCompany'
     - `getTeamMembers(locale)` → `TeamMember[]` — ordered by sortOrder, tag: 'teamMember'
     - `getTeamMemberBySlug(slug, locale)` → `TeamMember | null` — tag: 'teamMember'
     - `getFunds(locale)` → `Fund[]` — tag: 'fund'
     - `getNewsArticles(locale, limit?)` → `NewsArticle[]` — ordered by date desc, tag: 'newsArticle'
     - `getNewsArticleBySlug(slug, locale)` → `NewsArticle | null` — tag: 'newsArticle'
     - `getInvestmentPlatforms(locale)` → `InvestmentPlatform[]` — include 2 for fund references, tag: 'investmentPlatform'
     - `getInvestmentPlatformBySlug(slug, locale)` → `InvestmentPlatform | null` — tag: 'investmentPlatform'
     - `getPageBySlug(slug, locale)` → `Page | null` — tag: 'page'
     - `getSiteConfig(locale)` → `SiteConfig | null` — tag: 'siteConfig'
     - `getOffices(locale)` → `Office[]` — ordered by sortOrder, tag: 'office'
   - Batch approach: use `getEntries` with `limit: 100` (never per-entry fetches)
   - IMPORTANT: `unstable_cache` key arrays must include locale to avoid cross-locale cache hits

5. Create `src/lib/contentful/richText.tsx` following RESEARCH.md Pattern 6:
   - `'use client'` directive (uses React components)
   - Custom render options:
     - `BLOCKS.EMBEDDED_ASSET`: render with `next/image`, handle missing images gracefully
     - `INLINES.HYPERLINK`: render with `target="_blank" rel="noopener noreferrer"` for external links
     - `BLOCKS.HEADING_1` through `BLOCKS.HEADING_6`: semantic heading elements
   - Export `renderRichText(document: Document): React.ReactNode`

6. Create `src/app/api/revalidate/route.ts` following RESEARCH.md Pattern 5:
   - POST handler that:
     1. Validates `x-contentful-webhook-secret` header against `CONTENTFUL_WEBHOOK_SECRET`
     2. Returns 401 if invalid
     3. Extracts `sys.contentType.sys.id` from payload
     4. Calls `revalidateTag(contentTypeId)` — single argument (Next.js 15.5)
     5. For Asset changes (`sys.type === 'Asset'`), calls `revalidateTag('contentful-asset')`
     6. Returns `{ revalidated: true, contentType, now: Date.now() }`
   - Also export GET handler that returns `{ status: 'ok' }` for health check
