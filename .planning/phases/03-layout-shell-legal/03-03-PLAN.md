1. Create `src/components/cursor/CustomCursor.tsx` ('use client'):
   - Use `useRef` for the cursor DOM element — NO useState for position (avoids 60 re-renders/sec).
   - Use `useCallback` for the mousemove handler.
   - In `useEffect`:
     - Check `window.matchMedia('(hover: none)').matches` — if true (touch device), return early. No event listeners, no cursor element visible.
     - Add `mousemove` listener on `document` with `{ passive: true }`: update `cursorRef.current.style.transform = \`translate(\${e.clientX - 10}px, \${e.clientY - 10}px)\``.
     - Add `mouseover` listener on `document` (event delegation): check if `e.target` matches `'a, button, [data-cursor-hover], input, textarea, select'`. If yes, add class `cursor-hover` to cursor element.
     - Add `mouseout` listener on `document`: remove `cursor-hover` class.
     - Cleanup: remove all three listeners on unmount.
   - Render a styled div:
     - `position: fixed`, `top: 0`, `left: 0`, `width: 20px`, `height: 20px`, `border-radius: 50%`, `border: 2px solid var(--color-accent-teal)`, `pointer-events: none`, `z-index: 9999`, `mix-blend-mode: difference`, `will-change: transform`.
     - `transition: transform 0.15s ease, opacity 0.15s ease` — smooth scale transition on hover, but position updates are instant via JS (no transition on translate).
     - IMPORTANT: The transition should only apply to the scale transform, not the translate. To achieve this, use two nested elements: outer div positioned via JS (no transition), inner div with the visual circle and transition for scale. OR use a single element where the mousemove sets `left`/`top` directly (no transition) and the scale is applied via the `cursor-hover` class which has `transform: scale(2.5)` with transition.
     - Actually simplest approach: set position via `style.left` and `style.top` (instant, no transition), and use a CSS class for the scale effect with transition. The transform property on the cursor-hover class uses `!important` to override.
     - Correction for best approach: Use `transform: translate(Xpx, Ypx)` for positioning, and `transform: translate(Xpx, Ypx) scale(2.5)` when hovering. Store current x/y in a ref object, apply combined transform. The transition on transform will be fast enough (0.15s) that the lag from transition-delay on translate is minimal and actually looks smoother.
   - Hide on mobile: `display: none` by default, `display: block` only at `${mq.lg}`. Also `@media (hover: none) { display: none !important; }`.

2. Update `src/styles/GlobalStyle.ts`:
   - Add desktop cursor-hiding rules:
     ```css
     @media (hover: hover) and (pointer: fine) {
       body { cursor: none; }
       a, button, input, textarea, select, [data-cursor-hover] { cursor: none; }
     }
     ```
   - Add the cursor-hover class:
     ```css
     .cursor-hover {
       width: 50px !important;
       height: 50px !important;
     }
     ```
     Note: The scale approach is cleaner. Use whichever works best with the positioning method chosen above.
   - IMPORTANT: These rules hide the system cursor ONLY on devices with a mouse (hover: hover) and precise pointer (pointer: fine). Touch devices are unaffected.
