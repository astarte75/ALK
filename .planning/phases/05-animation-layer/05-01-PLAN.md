  <action>
Fix three critical bugs in LenisProvider.tsx:

1. **Add `autoRaf: false`** to the Lenis constructor options. Without this, Lenis runs its own internal RAF loop AND gets driven by GSAP ticker, causing a double RAF loop (jank/stutter).

2. **Fix ticker callback cleanup.** Current code passes an anonymous function to `gsap.ticker.add()` but tries to remove `lenis.raf` (wrong reference — silently fails, memory leak). Store the callback in a variable:
```typescript
const tickerCallback = (time: number) => {
  lenis.raf(time * 1000) // GSAP time is seconds, Lenis expects ms
}
gsap.ticker.add(tickerCallback)
// In cleanup:
gsap.ticker.remove(tickerCallback)
```

3. **Add `lenis.on('scroll', ScrollTrigger.update)`** — currently the scroll listener is a no-op comment. Without this, ScrollTrigger animations lag or don't fire during smooth scroll because Lenis intercepts native scroll events.

Also import `ScrollTrigger` from `@/lib/gsap-init` (currently only imports `gsap`).

Keep everything else identical: `duration: 1.2`, easing function, `touchMultiplier: 1.5`, `lagSmoothing(0)`, prefers-reduced-motion skip.

Replace ALL scattered `window.matchMedia('(prefers-reduced-motion: reduce)')` checks with `gsap.matchMedia()` inside each component's `useGSAP()` hook. This provides automatic cleanup when conditions change and unifies responsive + accessibility in one API.

**Pattern to apply in EVERY component:**

```typescript
useGSAP(() => {
  const mm = gsap.matchMedia()

  mm.add({
    isDesktop: '(min-width: 768px)',
    isMobile: '(max-width: 767px)',
    reduceMotion: '(prefers-reduced-motion: reduce)',
  }, (context) => {
    const { isDesktop, isMobile, reduceMotion } = context.conditions!

    if (reduceMotion) {
      // Set final states immediately — no animation
      // (component-specific gsap.set() calls)
      return
    }

    // Full animation code here (existing code, unchanged)
  })
}, { scope: containerRef })
```
