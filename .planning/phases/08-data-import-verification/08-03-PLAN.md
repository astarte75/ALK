# Phase 8 Plan 03: Admin Dashboard

## Objective
Build an admin dashboard for Alkemia staff to verify imported data, browse all investors and their positions, view all operations, and identify data inconsistencies.

## Tasks
1. Add `is_admin` column to investors table
2. Create `is_admin()` SQL function + admin RLS policies on all tables
3. Protect admin routes in middleware
4. Build admin dashboard page: summary cards, per-fund breakdown with totals, consistency checks
5. Build operations page: all capital calls with filters (fund, investor, type, date range)
6. Build investor impersonation: view dashboard and fund detail as any investor
7. Add admin link (gold accent) to PortalHeader
8. Fix investor dashboard to scope queries by investor_id (prevent admin RLS leak)
9. Add i18n translations (IT + EN)

## Success Criteria
- Admin can see all fund data, investor positions, and operations
- Consistency checks identify: investors without auth, without email, zero NAV positions, duplicates, empty records
- Operations filterable by fund, investor, type, date range with stable column widths
- Clicking an investor shows their exact dashboard view with fund drill-down
- Non-admin users redirected away from admin routes
