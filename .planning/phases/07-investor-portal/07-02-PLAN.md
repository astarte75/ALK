  <action>
Create `src/app/[locale]/investitori/layout.tsx`:
- This is the portal layout that REPLACES the site layout for portal routes
- Do NOT show the main site Header/Footer (portal has its own minimal chrome)
- Wrap children with `NextIntlClientProvider` (messages still needed)
- Include `StyledComponentsRegistry` if not inherited from root layout
- Add metadata: title "Area Investitori | Alkemia Capital" (IT) / "Investor Portal | Alkemia Capital" (EN)
- This layout applies to login page AND all protected sub-routes

Create `src/components/portal/PortalHeader.tsx`:
- Styled component: dark bar with Alkemia logo (link to `/`) and logout button (only when authenticated)
- Use CSS custom properties from existing theme (--color-bg, --color-surface, --color-border, --color-accent-teal)
- Logo: use `/images/alkemia-logo-white.png`
- Height: 64px, position sticky top 0, z-index above content
- Logout button: text "Esci" (IT) / "Sign Out" (EN) using `useTranslations`

Create `src/app/[locale]/investitori/actions.ts`:
- `'use server'` directive
- Export `login(prevState: any, formData: FormData)` server action:
  - Extract email and password from formData
  - Create server Supabase client
  - Call `supabase.auth.signInWithPassword({ email, password })`
  - On success: `redirect('/{locale}/investitori/dashboard')` (use revalidatePath first)
  - On error: return `{ error: 'Credenziali non valide' }` (or English equivalent based on locale)
  - Use `headers()` to detect locale from referer or accept-language
- Export `logout()` server action:
  - Create server Supabase client
  - Call `supabase.auth.signOut()`
  - Redirect to login page

Create `src/components/portal/LoginForm.tsx`:
- `'use client'` component
- Use React 19 `useActionState` (NOT `useFormState` which is deprecated) with the login server action
- Styled form with:
  - Alkemia logo centered above the form
  - "Area Investitori" / "Investor Portal" heading
  - Email input (type="email", required)
  - Password input (type="password", required)
  - Submit button styled with --color-accent-teal background
  - Error message display (from server action response)
  - Loading state on submit button (disable + show spinner)
- Premium dark aesthetic:
  - Centered card on dark background
  - Card: --color-surface background, --color-border border, subtle border-radius
  - Inputs: dark background (#1A1E22), light text, teal focus ring
  - Form max-width: 400px, centered vertically and horizontally
- All text from `useTranslations('portal')` for i18n
- Form action = login server action

Create `src/app/[locale]/investitori/page.tsx`:
- Server Component
- Check if user already authenticated: `const { data: { user } } = await supabase.auth.getUser()`
- If authenticated, redirect to `/investitori/dashboard`
- Otherwise render the LoginForm component
- Pass locale to LoginForm for proper action routing

IMPORTANT routing note: Next.js i18n with `localePrefix: 'as-needed'` means:
- IT (default): `/investitori` (no prefix)
- EN: `/en/investors`

The route files live at `src/app/[locale]/investitori/` but the EN path segment needs to be `investors`. Check if next-intl pathnames config handles this, or if a redirect/rewrite is needed. If next-intl supports `pathnames` config in routing.ts, add the mapping there:
```typescript
pathnames: {
  '/investitori': { it: '/investitori', en: '/investors' },
  '/investitori/dashboard': { it: '/investitori/dashboard', en: '/investors/dashboard' },
  // etc.
}
```
If next-intl v4 does not support this pattern with as-needed prefix, keep the path as `/investitori` for both locales (acceptable for v1 — the content is bilingual regardless of URL segment).
